# Задача 03: Управление шариком

## Приоритет: Высокий
## Оценка: Средняя
## Зависимости: Задача 02

## Описание
Реализовать управление шариком с помощью клавиатуры. Управление должно быть отзывчивым и использовать физические силы.

## Требования
- **КРИТИЧЕСКИ ВАЖНО:** Применять силы, а не менять позицию напрямую
- Стрелки влево/вправо - ускорение влево/вправо
- Стрелка вверх (или пробел) - прыжок (сила вверх)
- Обработка через keydown/keyup события (не текстовый ввод)
- Управление относительно камеры (не глобальное)
- Ускорение должно быть плавным пока кнопка нажата
- Шарик должен быть в 1.5 раза быстрее базовой скорости

## Критерии приемки
- [x] Управление работает резко, без задержек (силы навешиваются внутри `useBeforePhysicsStep`)
- [x] Шарик ускоряется при нажатии клавиш (анализируем состояние из `useInput`)
- [x] Прыжок применяет силу вверх (импульс `BALL_TUNING.jumpImpulse`)
- [x] Управление относительно камеры работает корректно (базисы вычисляются из `camera.getWorldDirection`)
- [x] Шарик достаточно быстрый для интересного геймплея (ограничения/макс. скорость вынесены в конфиг)

## Статус
Реализован хук `useBallController`, который привязывает ввод (WASD/стрелки + пробел) к импульсам и приводу относительно текущей камеры. Применение сил выполнено строго в детерминированном цикле Rapier, что исключает задержки и гарантирует повторяемость.
